# 博客更新与部署流程

本文档记录从撰写文章到部署上线的完整流程。

- 站点：<https://blog.gnix807.top>
- 仓库：<https://github.com/Gnix807/blog>
- 服务器项目目录：`/opt/blog-src`
- 网站运行目录（nginx 读取）：`/opt/1panel/www/sites/blog.gnix807.top/index`

---

## 一图流总结

```
写 .md 文章  →  (可选) pnpm dev 预览  →  git add/commit/push
                                              │
                          服务器: cd /opt/blog-src && bash deploy.sh
                                              │
                                    Ctrl+F5 看线上效果
```

---

## 1. 新建文章（本地）

文章放在 `content/posts/年份/` 下，**文件名即 URL**。

例如 `content/posts/2026/my-first-post.md` → `https://blog.gnix807.top/2026/my-first-post`

也可以用项目自带命令自动生成带模板的文件：

```bash
pnpm new
```

## 2. 撰写文章内容

每篇 `.md` 开头是 frontmatter（元信息）：

```markdown
---
title: 文章标题
description: 一句话摘要，利于 SEO
date: 2026-07-12 10:00:00
updated: 2026-07-12 10:00:00
image:                      # 封面图，可留空或放 public 下的图片路径，如 /cover.jpg
categories: [技术]          # 可选值见下
tags: [标签1, 标签2]
---

正文用 Markdown 撰写……
```

**可选分类**（在 `blog.config.ts` 定义）：`技术` / `开发` / `安全` / `杂谈` / `生活`

正文支持本主题的自定义组件，例如：

```markdown
::alert{type="tip" title="提示"}
这是一个提示框。
::

::quote
这是一段引用。
::

:copy{prompt="PS>" code="echo hello"}
```

## 3. 本地预览（可选，推荐）

```bash
pnpm dev
```

浏览器打开 <http://localhost:3000> 实时预览，改完按 `Ctrl+C` 停止。

## 4. 提交并推送到 GitHub（本地）

```bash
git add .
git commit -m "新增文章：文章标题"
git push
```

## 5. 部署到服务器

SSH 登录服务器，执行一键脚本：

```bash
cd /opt/blog-src && bash deploy.sh
```

`deploy.sh` 会自动完成：

1. `git pull` 拉取最新代码
2. `pnpm install` 安装依赖
3. `pnpm generate` 构建静态站点
4. 清空并拷贝产物到网站运行目录

## 6. 查看效果

浏览器强制刷新（`Ctrl+F5`）访问 <https://blog.gnix807.top>。

## 常见操作

### 修改配置 / 头像 / 导航等

与发文章流程完全一致：本地修改 → `git push` → 服务器 `bash deploy.sh`。

常用配置文件：

| 文件 | 作用 |
| --- | --- |
| `blog.config.ts` | 站点标题、作者、域名、统计/评论服务 |
| `app/app.config.ts` | 导航菜单、页脚社交链接、备案号等 |
| `app/assets/css/color.scss` | 主题配色（`--hue-theme` 色相） |
| `app/feeds.ts` | 友链列表 |
| `public/` | 头像、favicon、Logo 等静态资源 |

### 首次在新服务器部署

```bash
cd /opt
git clone https://github.com/Gnix807/blog.git blog-src
cd blog-src
bash deploy.sh
```

### 依赖报错时

若 `pnpm install --frozen-lockfile` 因依赖变动报错，去掉 `deploy.sh` 里的 `--frozen-lockfile` 再跑，或手动执行 `pnpm install`。
